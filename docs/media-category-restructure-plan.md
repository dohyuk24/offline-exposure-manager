# 매체 카테고리 4분류 + D-OOH 전용 뷰 — 기획

> **이 문서의 위치**: PRD 반영 전 단계의 기획 메모. 의사결정 후 PRD 흡수 + 구현.
> **작성일**: 2026-04-26
> **목적**: 4분류 모델(P/A/D/O OOH) 도입 + 배포형(D) 매체에 어울리는 뷰 설계

---

## 1. 4분류 모델 (사용자 정의)

| 코드 | 풀네임 | 정의 | v1 활성 |
|------|--------|------|---------|
| **P-OOH** | Paid OOH | 비용 지불 + 기간 단위 외부 광고 매체 | ✅ |
| **A-OOH** | Affiliated OOH | 비용 대신 혜택·관계·가치 교환으로 확보한 외부 매체 | ❌ (v2) |
| **D-OOH** | Distribution OOH | 단기·고빈도·노동 집약적 노출 (전단지·족자) | ✅ |
| **O-OOH** | Owned OOH | 제작비만으로 상시·반복 활용. 우리 통제 하의 매체 | ✅ |

### 관리 단위
- **P, O**: **지면별** (현재처럼 location_key 기반 카드)
- **D**: **배포 이벤트별** — 같은 위치에 여러 번 거는 게 아니라, "이 디자인을 OO일에 OO 지역에 N장 뿌렸다" 식
- **A**: v2 (현재 `media_type=바터제휴배너` 가 사실상 A 에 해당)

## 2. 현재 모델의 한계

### 카테고리
```typescript
MEDIA_CATEGORY = { OFFICIAL: '공식', OWNED: '자체보유', UNOFFICIAL: '비공식' }
```
- "공식" = P-OOH, "자체보유" = O-OOH 매핑 명확
- "비공식" 이 D-OOH(전단지·족자) + A-OOH(바터) + 기타 다 떠안고 모호
- `MEDIA_TYPE` 의 `BARTER_BANNER` 가 사실상 A-OOH 표시이지만, category 와 type 두 군데서 흩어져있음

### UI
- 지점 페이지 매체 그리드: "공식매체 (OOH)" / "비공식매체 · 자체보유" 2섹션
- 모든 카드가 동일한 모양 (사진 + 위치 + 상태)
- 전단지·족자도 일반 매체처럼 카드 1개 = 1 위치 → 배포 이력 누적이 안 됨
- "그동안 무슨 전단지를 어디에 뿌렸는지" 가 안 보임

## 3. 제안 — 카테고리 4분류 + 섹션 분리

### 3.1 매체 그리드 섹션 구조 (관리 탭)

```
[관리판 헤더 + ✨ 신규 매체 발굴 CTA]
[탭: 관리 · 현황 · 예산]
[오늘의 할 일 카드]

─── P-OOH (유가 옥외) ────  + 새 위치 발굴
  [카드 그리드 — 지면별]

─── O-OOH (자체 보유) ────  + 새 위치 등록
  [카드 그리드 — 지면별]

─── D-OOH (배포형) ───────  + 배포 기록
  [디자인 카드 그리드 — 디자인별]

─── A-OOH (제휴) ─────────  v2 예정
  [placeholder]
```

### 3.2 D-OOH 전용 카드 (제안)

배포형 매체는 "위치" 가 아니라 **"디자인"** 이 정체성. 같은 디자인을 여러 시기/장소에 반복 배포.

```
┌──────────────────────────────────┐
│ [전단지 사진]  봄 PT 프로모션 전단지 │
│              누적 5,300장 · 3회차  │
│              마지막 4/22           │
└──────────────────────────────────┘
```

클릭 → 배포 이력 타임라인 + "+ 새 회차 추가" 버튼:
```
4/22  강남대로 일대 · 1,500장
4/15  역삼역 출구 · 2,000장
4/01  학동사거리 · 1,800장
```

### 3.3 D-OOH 카드 단위 — 옵션 (결정 필요)

| 옵션 | 카드 = | 장점 | 단점 |
|------|--------|------|------|
| **a. 디자인 단위** | 1 디자인 + 회차 누적 | "무슨 전단지를" 한 눈에. 누적 체감 강함 | 디자인 동일성 판단 사용자 책임 |
| **b. 회차 단위** | 1 배포 이벤트 | 단순. 기존 매체 카드 패턴과 동일 | "같은 디자인 누적" 안 보임 |
| **c. 캠페인 단위** | 1 캠페인(예: 5월 봄) + 디자인·회차 묶음 | 마케팅 관점에 자연스러움 | 추가 entity 필요 |

> 추천: **a (디자인 단위)** — 사용자 표현 "무슨 전단지를 어디에" 와 직접 일치. v1 단순화 위해 캠페인 entity 는 도입 X. 디자인 동일성은 사용자 판단 (같은 디자인이면 같은 카드에 회차 추가).

### 3.4 D-OOH 카드 필드 (제안)

**카드 (= 디자인)**
- 디자인 사진 (전단지·족자 시안 1장)
- 디자인명 (예: "봄 PT 프로모션 전단지")
- 매체 종류 (`media_type`: 전단지 / 족자)
- 누적 수량 (자동 계산)
- 회차 수 (자동)
- 마지막 배포일

**회차 (= 배포 이벤트)**
- 배포일
- 배포지 (자유 텍스트, 예: "강남대로 일대")
- 수량 (장 / 매)
- 비용 (옵션, 인쇄/인건비)
- 메모

→ DB 스키마: `media_records` (디자인 카드) + 새 `distribution_events` 테이블 (회차)
또는 회차도 `media_records` 에 같은 location_key 로 묶어서 — 기존 히스토리 메커니즘 재활용

### 3.5 발굴 CTA 영향

- 현재 `/discover` 는 위치 라벨 + 사진 → P-OOH 발굴에 어울림
- D-OOH 는 발굴 패턴이 다름 (위치가 정체성 아님). "신규 발굴" 흐름이 D-OOH 에는 부자연스러움
- 제안: 헤더 CTA 는 그대로 P-OOH 발굴용 (`✨ 신규 매체 발굴`)
- D-OOH 추가는 D-OOH 섹션 옆 `+ 배포 기록` 버튼

## 4. 카테고리/타입 정리 (코드 레벨)

### 4.1 MEDIA_CATEGORY 변경
```typescript
// 변경 전
{ OFFICIAL: '공식', OWNED: '자체보유', UNOFFICIAL: '비공식' }

// 변경 후 (4개)
{
  PAID:        'P-OOH',  // 유가 옥외 (= 기존 공식)
  AFFILIATED:  'A-OOH',  // 제휴 (= 기존 비공식 중 바터)
  DISTRIBUTION:'D-OOH',  // 배포형 (= 기존 비공식 중 전단지/족자)
  OWNED:       'O-OOH',  // 자체 보유
}
```

### 4.2 MEDIA_TYPE 정리
- `BARTER_BANNER` → A-OOH 카테고리로 빠짐. type 으로는 일반 BANNER 와 동일. → `BARTER_BANNER` 삭제 가능 (v2 카테고리 도입 시)
- `LEAFLET`, `SCROLL` → D-OOH 카테고리 전용

### 4.3 마이그레이션 대상 (DB)
- 기존 `category='공식'` → `'P-OOH'`
- 기존 `category='자체보유'` → `'O-OOH'`
- 기존 `category='비공식'` → 분기 필요:
  - `media_type='전단지' OR '족자'` → `'D-OOH'`
  - `media_type='바터제휴배너'` → `'A-OOH'`
  - 그 외 → `'D-OOH'` 기본 (또는 사용자가 어드민에서 수정)

> 실데이터가 거의 없을 것 같으니 마이그레이션은 단순 SQL 1개로 처리 가능 (별도 PR 에서).
