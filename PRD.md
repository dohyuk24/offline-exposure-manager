# 오프라인 매체 관리 서비스 PRD

> v0.3 · Draft · 2026년 4월

---

## 1. 배경 및 목적

### 1.1 배경

현재 지점별 오프라인 매체 현황은 Notion 카드 단위로 관리되고 있으나 다음 문제가 있다.

- 매체 현황을 언제 어떻게 업데이트해야 하는지 구조화된 체계 없음
- 공식매체(OOH), 자체 보유 매체, 비공식매체가 혼재되어 카테고리별 현황 파악 어려움
- 마케팅실에서 전 지점 매체 현황을 한눈에 볼 수 없음
- 지점별 비공식매체 예산 잔액을 실시간 파악 불가
- 지점이 매체를 업데이트하거나 새로 발굴해도 서로 알 수 없어 동기부여 부족

### 1.2 목적

| # | 목표 | 대상 사용자 |
|---|------|------------|
| 1 | 옥외·비공식 매체 현황을 지점별로 체계적으로 관리 | 지점 담당자 / 마케팅실 |
| 2 | 지점별 비공식매체 예산 가시성 확보 (할당·사용·잔액) | 지점 담당자 / 마케팅실 어드민 |
| 3 | 업데이트와 신규 발굴 행위 자체에 의미와 피드백을 부여하여 자발적 참여 유도 | 지점 담당자 |

### 1.3 서비스 비전

- **지점 담당자:** "내가 오프라인 노출을 하고 싶을 때 이렇게 하면 되겠네! 기록하면 바로 반응이 와서 계속 하고 싶어진다."
- **마케팅실:** "지점에서 알아서 오프라인을 고민해주고, 하루 안에 전 지점 매체를 장악할 수 있다."

## 2. 사용자 및 역할

| 역할 | 설명 | 주요 권한 |
|------|------|----------|
| 지점 담당자 | 각 지점에서 매체 현황을 직접 관리하는 담당자 | 본인 지점 매체 현황 조회·등록·수정, 예산 잔액 확인 |
| 마케팅실 | 전 지점 현황을 모니터링하고 공식매체 의사결정을 담당 | 전 지점 현황 조회, 공식매체 아이디어 등록, 코멘트 |
| 어드민 (마케팅실 내) | 별도 비밀번호로 어드민 탭 접근 | 지점별 예산 설정·수정, 미션 설정, 전체 데이터 관리 |

**지점 목록 (초기 기준, 어드민에서 추가 가능)**
- 역삼ARC, 도곡, 신도림, 논현, 판교, 강변, 가산, 삼성, 광화문, 한티, 마곡, 판교벤처타운, 역삼GFC, 합정 등

## 3. 매체 분류 체계

| 카테고리 | 정의 | 예시 | 예산 귀속 |
|----------|------|------|----------|
| 공식매체 (OOH) | 유가 옥외 광고매체. 대행사를 통해 계약·집행 | 버스·지하철 광고, 빌보드 | 오피스 별도 예산 |
| 자체 보유 매체 | 지점이 상시 확보한 공간. 콘텐츠만 교체 | 외벽 현수막, 협의된 간판 | 지점 예산 (제작비) |
| 비공식매체 | 지점이 직접 발굴·집행하는 비유가 매체 | 바터제휴 배너/포스터, 현수막, 족자, 전단지 | 지점 월 할당 예산 |

**데이터 단위**
매체 레코드 1건 = "특정 월, 특정 매체에, 특정 콘텐츠로 노출된 건". 같은 위치라도 월이 바뀌면 새 레코드로 등록한다.

**매체 히스토리**
같은 위치의 매체는 레코드가 월별로 쌓이면서 자연스럽게 히스토리가 된다. 지점 페이지에서 최신순 정렬, 최근 업데이트 날짜, 과거 사진 등을 확인할 수 있다.

## 4. 정보 구조 (IA)

```
/
├── /                          홈 · 전체 현황 (마케팅실용 — 전 지점 요약)
│
├── /ranking                   점수판 · 랭킹 (전 지점 랭킹, 상위 3개 공개)
│
├── /branches
│   └── /branches/:branchId    지점 페이지 (해당 지점 것만)
│       ├── 공식매체 (OOH) 섹션
│       ├── 비공식매체 · 자체보유 섹션
│       ├── 예산 섹션
│       └── 이번 달 점수 섹션
│
├── /guide
│   ├── /guide/barter-bp        바터제휴 BP 아카이브 (공사 중)
│   └── /guide/media            매체별 가이드 (공사 중)
│
└── /admin                     어드민 (비밀번호 인증 · 마케팅실 전용)
```

**공통 UI — 발견 피드 배너**
모든 페이지 상단에 고정. 신규 매체 발굴 소식(좋은 소식)만 노출. 가장 최근 발굴 1건을 실시간으로 표시.

### 페이지별 접근 권한 및 주요 콘텐츠

| 페이지 | 접근 권한 | 주요 콘텐츠 |
|--------|----------|------------|
| 홈 · 전체 현황 | 전체 | 전 지점 현황 요약 카드 (활성 매체 수, 업데이트 여부, 예산 소진율) |
| 점수판 · 랭킹 | 전체 (상위 3개) / 어드민 (전체) | 전 지점 랭킹. 1~3위 공개, 이하 어드민에서만 확인 |
| 지점 페이지 | 해당 지점 담당자 / 마케팅실 | 해당 지점의 공식·비공식 매체, 예산, 점수 (탭 없이 단일 스크롤) |
| 바터제휴 BP | 전체 | 우수 바터제휴 사례 아카이브. 콘텐츠 형식 (유저 인터뷰 스타일). 공사 중. |
| 매체별 가이드 | 전체 | 현수막·족자·전단지 제작 가이드. 공사 중. |
| 어드민 | 마케팅실 전용 | 전체 랭킹, 예산 설정, 점수 기준, Slack 설정, 지점 관리 |

## 5. 핵심 기능

### 5.1 매체 현황판

#### 5.1.1 지점 페이지

- 각 지점마다 독립적인 페이지 `/branches/:branchId`
- 탭 없이 단일 스크롤 — 공식매체 / 비공식매체 / 예산 / 점수 순서로 배치
- 지점 담당자는 고유 토큰 URL로 로그인 없이 진입
- 매체 레코드는 최신순 정렬, 최근 업데이트 날짜 표시
- 마케팅실은 전 지점 페이지 자유 열람 가능

#### 5.1.2 매체 레코드 필드

| 필드명 | 설명 | 입력 방식 | 필수 |
|--------|------|----------|------|
| 구분 | 공식 / 자체보유 / 비공식 | 드롭다운 | ✅ |
| 매체 종류 | OOH / 현수막 / 족자 / 전단지 / 바터제휴 배너 / 기타 | 드롭다운 | ✅ |
| 상태 | 협의 중 / 집행 중 / 종료 / 보류 | 드롭다운 | ✅ |
| 지점 | 해당 지점명 (페이지 진입 시 자동 세팅) | 자동 | ✅ |
| 위도 (latitude) | 매체 위치 GPS 좌표 | 숫자 입력 또는 GPS 자동 수집 | ✅ |
| 경도 (longitude) | 매체 위치 GPS 좌표 | 숫자 입력 또는 GPS 자동 수집 | ✅ |
| Zone ID | 상권 구역 식별자 (추후 지도 연동 대비) | 드롭다운 또는 자동 매핑 | ✅ |
| 유형 | 이미지 / 텍스트 / 영상 | 드롭다운 | - |
| 규격 / 사이즈 | 예: 가로 2m × 세로 1m | 텍스트 | - |
| 설명 | 매체 추가 설명, 위치 정보 등 | 텍스트 | - |
| 날짜 | 노출 기간 (시작일 ~ 종료일) | 날짜 선택 | - |
| 사진 첨부 | 현재/과거 머티리얼, 현장 사진 | 인앱 카메라 호출 또는 파일 업로드 | - |
| 비용 | 집행 비용 (비공식·자체보유만 해당) | 숫자 입력 | - |
| 바터제휴 조건 | 바터제휴 매체인 경우 조건 기재 | 텍스트 | - |
| 고유번호 (마케팅팀) | 내부 관리용 코드 | 텍스트 | - |
| 마케팅팀 관리 | 마케팅실 관리 여부 | 체크박스 | - |

> **위도·경도·Zone ID 비고:** 현재 버전에서는 필드를 DB에 포함하여 저장만 한다. 지도 시각화 및 상권 분석 기능은 추후 개발한다. GPS 자동 수집은 모바일 브라우저의 Geolocation API를 활용하며, 사용자가 허용 시 자동 입력된다.

#### 5.1.3 공식매체 (OOH) 현황 및 제안

- 지점이 상권 순회 후 공식매체 후보를 사진과 함께 제안
- 제안 등록 시 마케팅실 담당자에게 Slack 즉시 알림 발송
- 현재 집행 중인 공식매체의 계약 조건, 대행사 정보 기재
- 오피스가 코멘트로 의사결정 진행 (승인 / 보류 / 킬)
- 예산 항목은 공식매체에 포함하지 않음 (오피스 별도 관리)

### 5.2 예산 관리

| 항목 | 내용 |
|------|------|
| 예산 단위 | 지점별 · 월별 독립 예산 |
| 기본 할당액 | 월 500,000원 (어드민에서 지점별·기간별 수정 가능) |
| 사용 대상 | 비공식매체 제작·집행 비용, 자체 액션 관련 비용 |
| 예산 외 항목 | 공식매체(OOH)는 오피스 별도 예산으로 관리 (본 서비스 미포함) |

- 지점 담당자: 이번 달 할당 / 사용 / 잔액 실시간 조회
- 매체 카드에 비용 입력 시 사용액에 자동 반영
- 마케팅실: 전 지점 예산 현황 요약 조회 (홈)
- 어드민: 지점별 월 할당 예산 설정 및 수정

### 5.3 참여 피드백 시스템

> **핵심 루프:** 상권 순회 → 매체 발견/업데이트 → 즉시 기록 → 앱이 반응 → 전체 피드에 공유 → 점수 반영 → 다음에도 기록하고 싶어짐

#### 5.3.1 업로드 완료 즉시 마이크로 피드백

단순 "저장 완료" 토스트가 아니라, 행위 자체에 의미를 부여하는 반응.

- **업데이트 시:** "역삼 현수막 업데이트 완료! 이번 달 3번째 기록이에요 🎉"
- **신규 발굴 등록 시:** "새 매체 발굴! 판교점 이번 달 첫 신규 확보예요 ✨"
- **바터제휴 성사 시:** "바터제휴 성사! 지점 예산 아끼면서 노출 확보했어요 👍"

피드백 문구는 매체 종류 + 이번 달 누적 업데이트 횟수 + 신규 발굴 여부를 조합하여 동적으로 생성.

#### 5.3.2 발견 피드 배너 (전 페이지 공통)

- 모든 페이지 상단에 고정 노출
- 신규 매체 발굴 소식(좋은 소식)만 표시. 일반 업데이트는 미포함.
- 형식: "[지점명]이 새 [매체 종류]를 확보했어요! 👀" + 시간
- 다른 지점 담당자 및 마케팅실이 해당 지점 페이지로 바로 이동 가능

#### 5.3.3 활동 점수판 및 랭킹

| 활동 | 점수 | 비고 |
|------|------|------|
| 기존 매체 현황 업데이트 | 1점 | 사진 포함 시 +0.5점 |
| 신규 매체 발굴 등록 | 5점 | 발견 피드에도 공유됨 |
| 바터제휴 성사 등록 | 7점 | 제휴 조건 기재 시 유효 |

- 점수 기준은 어드민에서 조정 가능
- 전 지점 랭킹: 상위 3개 지점은 `/ranking` 페이지에 공개, 이하는 어드민에서만 확인
- 지점 담당자는 본인 지점의 이번 달 점수 + 순위를 지점 페이지 상단에서 항상 확인
- 월 단위 리셋 (누적 점수는 별도 보관)

### 5.4 운영 워크플로우 — Slack 연동

Slack Incoming Webhook 또는 Slack API로 구현.

#### 5.4.1 승인 프로세스 알림 (마케팅실 수신)

| 이벤트 | Slack 메시지 | 수신자 |
|--------|-------------|--------|
| 공식매체 후보 제안 등록 | "[역삼ARC] 공식매체 후보 제안이 올라왔어요. 검토해주세요 → [링크]" | 마케팅실 담당자 |
| 기획 지원 요청 (추후) | "[판교] 디자인 제작 요청이 접수됐어요 → [링크]" | 마케팅실 담당자 |
| 신규 매체 발굴 등록 | "[판교] 새 현수막 위치 확보! 사진 첨부 → [링크]" | 마케팅실 채널 |
| 바터제휴 성사 등록 | "[역삼ARC] 바터제휴 성사 — [업체명]과 [조건] → [링크]" | 마케팅실 채널 |

#### 5.4.2 리마인더 알림 (지점 수신)

| 트리거 | Slack 메시지 | 수신자 |
|--------|-------------|--------|
| 월 N일 이후 업데이트 없음 | "[도곡] 이번 달 매체 현황 업데이트가 아직 없어요. 지금 기록하러 가기 → [링크]" | 해당 지점 |
| 월말 예산 70% 미소진 | "[신도림] 이번 달 예산 [N원] 남았어요. 비공식매체 활용 기회 있으면 기록해주세요." | 해당 지점 |

- 알림 발송 기준일(N일)은 어드민에서 설정
- 지점별 Slack 채널 또는 DM 대상은 어드민에서 매핑

## 7. 데이터 구조 비고

### 7.1 매체 레코드 DB 필드 (GPS · Zone ID)

매체 레코드 DB 설계 시 다음 필드를 필수값으로 포함한다.

```
media_records
├── id
├── branch_id
├── category          (공식 / 자체보유 / 비공식)
├── media_type         (OOH / 현수막 / 족자 / ...)
├── status             (협의중 / 집행중 / 종료 / 보류)
├── latitude           ← 위도 (GPS, 추후 지도 연동 대비)
├── longitude          ← 경도 (GPS, 추후 지도 연동 대비)
├── zone_id            ← 상권 구역 ID (추후 지도 연동 대비)
├── content_type
├── size
├── description
├── start_date
├── end_date
├── cost
├── barter_condition
├── internal_code
├── managed_by_marketing  (boolean)
├── photos[]           (url array)
├── created_at
├── updated_at
└── deleted_at         (소프트 딜리트)
```

> `latitude`, `longitude`, `zone_id`는 현재 버전에서 저장만 하고 지도 시각화에는 활용하지 않는다. 추후 상권 분석, 지도 기반 매체 뷰 개발 시 활용한다.

## 10. 용어 정의

| 용어 | 정의 |
|------|------|
| OOH (공식매체) | Out-of-Home. 버스·지하철 등 유가 옥외 광고매체 |
| 자체 보유 매체 | 지점이 이미 계약·확보한 상시 노출 공간. 콘텐츠만 교체 |
| 비공식매체 | 지점이 직접 발굴·집행하는 비유가 매체 |
| 바터제휴 | 현금 거래 없이 상호 서비스를 교환하여 확보하는 노출 공간 |
| BP (Best Practice) | 선임지점장 회의 등에서 공유되는 우수 사례 |
| 머티리얼 | 매체에 실제로 노출되는 디자인 결과물 |
| 신규 발굴 | 기존에 없던 새로운 위치나 채널에서 처음 매체를 확보하는 행위 |
| 마이크로 피드백 | 업로드 직후 노출되는 맥락형 응원 메시지 |
| Zone ID | 상권 구역 식별자. 추후 지도 연동 및 상권 분석에 활용 예정 |
| 어드민 | 마케팅실 전용 관리자 탭. 별도 비밀번호로 접근 |
